本地进行进行的编辑
0.配置git
	git config 命令
	git config --list 显示当前git的配置信息
	git config -e 配置当仓库的信息
	git config -e --global  针对系统上的所有仓库

设置提交代码时的用户信息
	git config --global user.name "xxx"
	git config --global user.email xxx@xxx.com
如果去掉--global参数只对当前仓库有效

1.本地进行git的创建
	选择一个目录作为git的仓库，只需要用git init 命令来初始化它
	或者指定一个目录作为git仓库，用git init newrepo

2.初始化完成后，在目录下会出现一个.git的目录，存放所有的git需要的数据和资源。
	然后讲当前目录下的几个文件加入版本控制库中，需要用git add 命令告诉git开始对这些文件追踪
	然后用 git commit -m ‘初始化项目版本’。
	如果前面没有添加远程仓库，则需要添加远程仓库：如：git remote add origin git@github.com:xxx/xxx.git 

3.还可以使用git clone 从现有git仓库中拷贝项目。 如：git clone <repo>
	如果需要克隆到指定目录。 如：git clone <repo> <directory>
	如果拷贝其他人的项目。如 git clone git://github.com/xxx/xxx.git 
	要定义新建项目的名称。可以在命令末尾指定新名字。
/***********在远程上进行改动的操作
4.查看当前的远程库   git remote  加上-v参数看，可以看到每个别名的实际链接地址
5.提取远程仓库
	使用git fetch origin 从远程仓库下载新分支到你所在的分支
	从远端仓库提取数据并尝试合并到当前分支 git merge origin/master
6.推送到远程仓库
	git push origin master
7.删除远程仓库
	git remote rm [别名]
